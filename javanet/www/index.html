Welcome to the cajo distributed object framework.

 The cajo project is a small free library enabling powerful dynamic multi-machine cooperation, both within, and between Java applications. It provides an easy-to-use, yet completely understandable framework to simplify the use of RMI, while at the same time harnessing its full potential. It is being successfully deployed on J2EE, J2SE, and J2ME platforms.

Project Goals


1. Establish a free standard distributed paradigm, which becomes increasingly valuable with greater use.

2. Encourage increased distributed inter-operation between applications and components.

3. Foster development of scale-free component networks; creating flexible resilient and adaptive applications.

Major Features


1. The framework allows any number of Java Virtual Machines to work together, literally as one. Application code makes absolutely no distinction between local and remote objects. Applications can evolve from all objects being on a single Virtual Machine, to all objects being on separate Virtual Machines, to all objects being on separate Physical Machines. The scalability comes with the framework, it requires no changes to the application's source code.

2. Unsigned applets, and Web Start applications, indistinguishable from locally installed applications in terms of interactivity, can be created easily. Transfer an application's entire user interface to any Java equipped client. Cleanly separate the computational components of the model, view, and controller portions of an application between multiple machines. Send both data and code, with complete transparency.

3. The framework allows both applications, and developers, to collaborate dynamically, linking their objects at will. This architecture can fulfill the true promise of Java: turn the network into one seamless evolving computer; linking everyting from mainframes, to mobile phones. It can allow applications to be built using optimized components from other distributed applications.

Getting started


The framework consists of just two small packages; gnu.cajo.invoke (5 classes) defining the architectural foundation, and gnu.cajo.utils (8 classes) applying the foundation to create the framework.

There are two general approaches to getting started:

1. For those who want to fully understand the implementation of this framework, first review the invoke package, it's surprisingly small. Next move on to the utils package. This will make the provided example code both immediately understandable, and quickly extensible. It's also an excellent way to learn about RMI in general. A theory page has been provided, especially for this group.

2. For those who want to jump right in, and learn by doing, there is the example package consisting of only four small classes, yet completely exercising the entire framework. It fully demonstrates the ease of use of this framework. The project's root directory has batch scripts illustrating three fundamentally different ways to build the application.

Either way, after downloading all the free source code, the next step would be to read the tutorial, then review the component articles on the right sidebar.

For the CVS challenged; a zipped snapshot of the complete project codebase is located here. (62kB, 12-Feb-04) I'll manually keep it up to date. However, by not using CVS you'll have to periodically check back to see if it has changed. You can bookmark it though, or link to it, as I'll keep its URL (https://cajo.dev.java.net/cajo.zip) constant.

Special Announcements


I hope you find it surprisingly easy to come up to speed. I also hope you will find the new possibilities very exciting. The framework is lean and clean, and this project is not about making it larger. Rather, its focus is on promoting its adoption into new, and existing, applications. Naturally, improvements and fixes are always welcome, however they must be backward compatible; to maintain interoperability with existing applications.

Once our member community reaches critical mass, I hope this site will become a thriving resource for developers. Until then, I will do my best to handle any of your questions, comments, and ideas. I recommend using the project forums, as I currently subscribe to all of them. Migrate some of your favorite code, and let's hear about the results. The news section will host the latest information about the project.

*Most importantly, if you want to really show your support for this project; please become a member. (It's also free!) Our membership enhances this project's status as a public standard. Feel free to post a hello in the introduction forum if you like, and say a little about yourself.
