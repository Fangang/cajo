Welcome to the cajo project
IANA port 1198, IANA UDP address 224.0.23.162

The cajo project is a small, free library, enabling powerful dynamic multi-machine coöperation; both within and between, both free and proprietary Java applications. It provides a surprisingly easy to use, yet completely understandable framework to dramatically simplify the use of RMI; whilst at the same time harnessing its full potential. It is actively being deployed in Java EE, SE, and ME environments; all over the world. It is a uniquely 'drop-in' distributed computing framework: meaning it imposes no structural requirements on your applications, nor source changes. It allows multiple remote JVMs to work together seamlessly, as one. Welcome: To the home of the World Wide Virtual Machine.

Project goals

1. Establish a free, simple, distributed paradigm; which becomes increasingly valuable with greater adoption.

2. Provide a clean, modular, and completely testable approach; for distributed application development.

3. Encourage increased distributed inter-operation; between developers, applications, and component libraries.

Major features

1. The framework allows any number of Java Virtual Machines to work together, literally as one. Applications can evolve dynamically; from all objects being on a single Virtual Machine, to all objects being on separate Virtual Machines, to all objects being on separate physical Machines -- even at runtime.

2. Easily create completely safe, unsigned thin-client applications, both as Applets and via WebStart, which are indistinguishable from locally installed applications. Transfer an application's entire user interface, to any Java equipped client, automatically. Cleanly separate the components of the model, the view, and the controller portions of an application between multiple machines. Send both data and code, with complete transparency, and security.

3. This framework allows both applications, and developers, to collaborate dynamically, linking their objects at will. This architecture can literally turn the network into one, seamless evolving computer; linking everything from mainframes, to mobile phones. It can allow applications to be easily built using optimised components from other distributed applications. Remoting and linking of objects can be done with as little as one additional line of code.

This framework has no dependencies on any 3rd party libraries. It works with all Java Runtime Environments, 1.2 and higher; this is especially important, to include JME/embedded devices. Coöperating Virtual Machines can be behind NAT, firewalls, even HTTP proxies.

Best of all, this framework is 100% pure Java TM: i.e. it uses no bloody <XML>!
(nor any silly @nnotations, neither)

The core framework consists of just three very small, and well documented packages:
  • gnu.cajo (2 classes) provides a high-level functional interface
  • gnu.cajo.invoke (6 classes) defines the architectural core for clients
  • gnu.cajo.utils (8 classes) applies this core for the creation of servers

Getting started

The recommended Quick Start, is to read the short overview page; then try some of our short experiments:
  • First steps
  • The grail
  • JVM scripting
  • Object transparency
  • Using proxies
  • Reference multicasting
Following that, there are two general approaches:

A. For those who want to fully understand the implementation of this framework;
  • First review the invoke package, it is surprisingly small
  • Next move on to the utils package
  • A theory page has been provided, especially for this group
This will make the provided example code both quickly understandable, and readily extensible. It's also an excellent way to learn about RMI in detail.

B. For those who want to jump right in, and learn by doing;
  • Look at the example, it consists of four small classes, yet exercises the entire framework
  • The tutorial page, should be of particular interest to this group
The project's root directory has both batch files, and a complete ant script, illustrating four strategically different ways to build the example.

The discussion fora are an interesting and valuable resource. The Documents & files section holds optional extension classes, and serves as a public developement area. There is also a wiki, where members can offer examples, clarify details, or supplementally create any additional project documentation.

Note: links to all this information, and much more, can be found on the left sidebar.

Community mission

We hope you find it surprisingly fast and easy to come up to speed. We also hope you find the new possibilities very exciting. This framework is both clear and focused. We are not interested in senselessly making it larger. Rather, our primary goal is promoting and assisting its adoption into new and existing applications. Of course, improvements and fixes are always welcome; from everyone! However, the interesting challenge is: All changes must be both backward and forward compatible, in order to maintain interoperability with currently running applications. E.g. consider the public telephone network: A 1920's-era rotary phone can call the newest satellite phone, on the other side of the planet, and vice versa.

Most importantly: If you would like to show your support, please consider becoming a project Observer. Our membership strengthens our framework's position, as a free public standard. It also encourages more developers to look into our codebase. Please feel free to post a hello in the introduction forum if you wish, as well as to make use of every other support forum. In the spirit of global community; you can even give your home continent its membership representation.
