Welcome to the cajo distributed object framework.

 The cajo project is a small, free library enabling powerful, dynamic cooperation between Java applications. It provides an easy-to-use, openly understandable framework to simplify the use of RMI, while harnessing its full potential.

Goals of the project:
  • Establish a standard paradigm, which becomes increasingly valuable with greater use.

  • Encourage increased dynamic inter-operation between applications.

  • Foster development of scale-free application networks; where functionality is flexible, resilient, and adaptive.

  • Encourage the development of subprojects! Everything from CaJo Pong to CaJo Quake, from CaJo Instant Messager to CaJo P2P, and many other interesting things. (Of course you don't really have to name them that way)

Essentially, this package allows an unlimited number of Java Virtual Machines to work together as one. What's more is the application code makes no distinction whatsoever between local activity and remote. Both machines, and developers, can collaborate dynamically; coming and going at will, fulfilling the promise of Java to turn the network into one seamless computer!

How to get started:

The framework consists of just 2 small packages; gnu.cajo.invoke (5 classes) and gnu.cajo.utils (9 classes).

For those who want to learn the ins-and-outs of the internal design; the invoke package defines the core paradigm and is the best place to start. The utils package applies the paradigm to create the framework.

For those who want to jump right in, and learn by doing, there is an example package of 4 small classes fully exercising the framework. The project's root directory has batch scripts illustrating 3 fundamental ways to build the application. There is also a doc batch, to create an exhaustive (literally!) javadoc site in a subdirectory named docs.

Either way, after downloading all the source, the next best step would be to visit the tutorial.
