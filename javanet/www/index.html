Welcome to the cajo distributed object framework.

 The cajo project is a small, free library, enabling powerful, dynamic, multi-machine interaction, both within, and between, Java applications. It provides an easy-to-use, yet completely understandable framework to simplify the use of RMI, while at the same time harnessing its full potential. It has been successfully tested and deployed on J2EE, J2SE, and J2ME platforms.

Goals of the project:
  1. Establish a standard distributed paradigm, which becomes increasingly valuable with greater use.

  2. Encourage increased distributed inter-operation between applications and components.

  3. Foster development of scale-free component networks, to create flexible, resilient, and adaptive applications.
1. The framework allows an unlimited number of Java Virtual Machines to work together as one. The application code makes absolutely no distinction between local and remote invocations. Applications can evolve from all objects being on a single Virtual Machine, to all objects being on separate Virtual Machines, to all objects being on separate Physical Machines. The scalability comes with the framework, requiring no changes to the application's source code.

2. Unsigned applets, indistinguishable from applications in terms of interactivity, can be easily created. Transfer an application's entire user interface to any Java capable browser. Cleanly separate the computational components of the model, view, and controller portions of an application, between multiple Virtual Machines. Send both data, and code, with complete transparency.

3. The framework allows both applications, and developers, to collaborate dynamically, linking their objects at will. This architecture can fulfill the real promise of Java: to make the network into one seamless, evolving computer; linking everyting from mainframes to mobile phones. It can allow applications to be built using optimized components from many other distributed applications.

Getting started:

The framework consists of just two small packages; gnu.cajo.invoke (5 classes) defining the architectural foundation, and gnu.cajo.utils (8 classes) extending and applying the foundation to create the framework.

1. For those who want to fully understand the implementation of the framework; first review the invoke package, it's surprisingly small. Next move on to the utils package. This will make the provided example code both completely understandable, and quickly extensible. It's also an excellent opportunity to learn about RMI in general.

2. For those who want to jump right in, and learn by doing, there is an example package consisting of just four small classes, yet completely exercises the entire framework. I'ts very easy to extend. The project's root directory has batch scripts illustrating three fundamentally different ways to build the application.

Either way, after downloading all of the source, the next step would be to read the tutorial, then review the component articles on the right sidebar.

I hope you find it surprisingly easy to come up to speed. I am confident you will find this more than worth the effort. The framework is lean and clean, and this project is not about making it larger; rather its focus is on promoting its adoption into new and existing applications. Of course improvements and fixes are always welcome, however they must be backward compatible.

Once our developer community reaches critical mass, I hope this site will become a thriving user support resource.
