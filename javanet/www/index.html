Welcome to the cajo project

The cajo project is a small, free library, enabling powerful dynamic multi-machine cooperation, both within, and between, Java applications. It provides an easy-to-use, yet completely understandable framework to simplify the use of RMI, while at the same time harnessing its full potential. It is being successfully deployed on J2EE, J2SE, and J2ME environments. It is particularly easy to add to existing programs, as it requires no architectural redesign.

Project Goals

1. Establish a free simple distributed paradigm, which becomes increasingly valuable with greater use.

2. Encourage increased distributed inter-operation between applications, and component libraries.

3. Foster development of scale-free network components; creating flexible, resilient, and adaptive applications.

Major Features

1. The framework allows any number of Java Virtual Machines to work together, literally as one. Application code makes absolutely no semantic distinction, between local, and remote objects. Applications can evolve from all objects being on a single Virtual Machine, to all objects being on separate Virtual Machines, to all objects being on separate Physical Machines. The scalability comes via the framework; it requires no changes to the application's code, or its structure.

2. Unsigned applets, and Web Start applications, indistinguishable from installed applications, can be created easily. Simply transfer an application's entire user interface, to any Java equipped client. Cleanly separate the computational components of the model, view, and controller portions of an application, between multiple machines. Send both data and code, with complete transparency, and security.

3. This framework allows both applications, and developers, to collaborate dynamically, linking their objects at will. This architecture can fulfill the true promise of Java: Turn the network into one seamless, evolving computer; link everything, from mainframes, to mobile phones. It can allow applications to be easily built, using optimised components from other distributed applications. Remoting and linking of objects can be done with as little as one additional line of code!

Getting Started

The framework consists of just two, very small packages:
  • gnu.cajo.invoke (5 classes) defines the architectural core for clients
  • gnu.cajo.utils (8 classes) applies this core for the creation of servers
The project has no dependencies on any other frameworks, and works with any Java Runtime Environment; 1.2 or higher. This is critically important, to include J2ME devices. Best of all, it uses: No bloody XML!TM

In general; there are two approaches to getting started:

1. For those who want to fully understand the implementation of this framework, first review the invoke package, it's surprisingly small. Next move on to the utils package. This will make the provided example code both immediately understandable, and quickly extensible. It's also an excellent way to learn about RMI, in detail. A theory page has been provided, especially for this group.

2. For those who want to jump right in, and learn by doing, there is the example package. It consists of just four small classes, yet completely exercises the entire framework, and demonstrates its ease of use. The project's root directory has batch files, and an Ant script, illustrating three fundamentally different ways to build the application. The overview page should be of particular interest to this group.

Note: links to all this information, and much more, can be found on the left sidebar, under Project tools.

Either way, after downloading the free source code, the next step would be to read the tutorial.

Special Announcements


I hope you find it surprisingly easy to come up to speed. I also hope you will find the new possibilities very exciting. This framework is lean and clean, and this project is not focused on making it larger. Rather, its primary focus is on promoting its adoption into new, and existing, applications. Naturally, improvements and fixes are always welcome, from everyone. However, they must be backward compatible, to maintain interoperability with existing applications.

Once our member community reaches critical mass, I hope this site will become a popular gathering place for developers. Until then, I will do my best to handle any of your questions, comments, or ideas. I recommend using the project's fora, as I currently subscribe to all of them. Migrate some of your favorite code, and let's hear about the results. The Announcements section will host the latest information about the project, and the Documents & files section will hold optional extension classes, and serve as a public developement area. We also have a wiki section, where members can offer examples, clarify details, or create any additional project documentation.

*Most importantly; if you want to really show your support for this project, please become an Observer. (It's also free) Our membership strengthens our project's position, as a free, public, standard. You are then welcomed to post a hello in the introduction forum if you like, and say a little about yourself.
