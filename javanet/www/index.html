Welcome to the cajo distributed object framework.

 The cajo project is a small, free library enabling powerful, dynamic cooperation between Java applications. It provides an easy-to-use, openly understandable framework to simplify the use of RMI, while harnessing its full potential.

Goals of the project:
  • Establish a standard paradigm, which becomes progressively more valuable as more developers use it.

  • Encourage increased practice of dynamic inter-operation between applications.

  • Foster development of scale-free application networks, where functionality is flexible, resilient, and adaptive to loading.

  • Encourage the development of subprojects! Everything from CaJo Pong to CaJo Quake, from CaJo Instant Messanger to CaJo P2P, and lots of other interesting things. (Of course you don't really have to name them that way! :-)

The framework consists of just 2 small packages; gnu.cajo.invoke (5 classes) and gnu.cajo.utils (9 classes).

For those who want to learn the ins-and-outs of the internal workings, the invoke package defines the paradigm, and is the best place to start. The utils package applies the invoke package paradigm to create the framework.

For those who like to jump right in, and learn by doing, there is an example directory of 4 small classes fully exercising the framework. The project's root directory has batch scripts illustrating 3 fundamental ways to build the application. There is also a doc batch, to create a full javadoc site in a subdirectory named docs.
