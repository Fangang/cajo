About the project licensing...

 This framework is licensed under the GNU LGPL, the technical legalese can be found here. The short, human-readable preamble is definitely worth reading.

Let me summarise the two most important parts:
  • This license grants you the right to use this code in your applications, without payment of any royalties to the copyright holder; he being I.

  • In a feature not present in the regular GPL; you have the right to use this library in proprietary applications, without having to make the combined code free.
Many will argue that this second point makes the license "less free" than the GPL; in fact I also believe that it does. However, this framework has its greatest opportunity to influence application development if it is a resource for all developers, not just the free ones. Just as with its objects, this framework allows all of its developers to work together; whether they have the option to make their code free or not.

This license does impose an important provision however; which in order to use this code, you must also agree:
If you modify the gnu.cajo classes or the interfaces, or the packages; if you add classes or interfaces to the packages, or create subpackages: The resulting packages must also be provided under the same terms of the GNU LGPL, or if you prefer, the GNU GPL. However; extending the classes or implementing the interfaces in different packages, is not subject to this provision.
