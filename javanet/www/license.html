About the package licensing.

 This package is licensed under the GNU LGPL, the technical legalese can be found here. I will try to summarize it, here. The license grants you the right to use this code, as you wish, without any payment of any royalties to the author; he being me. Further, in a feature not present in the regular GPL, you can use this library in a proprietary application, without having to make the resulting code free.

Some people will argue that this makes the license "less free" than the GPL, and perhaps it is. However I don't believe software is really free, if its license also tells you how you can't use it. This framework has its greatest chance to influence application development if it is a resource for all developers, not just the "free" ones. I like to think of this framework as a bridge, between free and non-free software. Just as with the objects, the framework lets all developers work together, whether they make their code public or not. /rant

Don't get me wrong, I love the GPL, and I think it is the perfect license for applications, especially operating systems. For libraries, however, I wish rms would have left the original acronym, meaning "Library General Public License" rather than changing it to "Lesser General Public License", but strictly limiting its use only for libraries.

BTW: I would apprecitate if you distributed the framework source with your applications, or at least a link back to this site. But to allow you ultimate freedom; I'll even leave that up to you.

A free library, so free it can be used for free, in free and non-free, applications, freely.
