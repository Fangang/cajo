About the project licensing...

 This framework is licensed under the GNU LGPL, the technical legalese can be found here. The short, human-readable preamble is definitely worth reading.

Let me summarise the two most important parts:
  • The license grants you the right to use this code in your applications without payment of any royalties to the copyright holder; he being me.

  • In a feature not present in the regular GPL; you are free to use this library in proprietary applications, without having to make the combined code free.
Many people will argue that this second point makes the license "less free" than the GPL, in fact I also believe it does. However, this framework has its greatest opportunity to influence application development if it is a resource for all developers, not just the free ones. Just as with its objects; this framework also allows its developers to work together, whether they have the option to make their code free or not.

The license imposes one important provision however; in which to use this code you must also agree:
If you add any classes to the gnu.cajo packages; or rename them, or modify them in any other way:
These resulting packages are required to be licensed under the GNU LGPL.
This provision is simply to keep this code, and its future enhancements, forever free.
